th要素が1つしかない例

テーブル内にth要素が1つしかなく、行見出しなのか列見出しなのか判別が難しい場合に、scope属性を用いると恩恵があるのではないか。

Contact Information
Name Phone# Fax# City
1. Joel Garner 412-212-5421 412-212-5400 Pittsburgh
2. Clive Lloyd 410-306-1420 410-306-5400 Baltimore
3. Gordon Greenidge 281-564-6720 281-511-6600 Houston

上のテーブルでは、1行目の4列目「City」にのみ th scope="row" が指定されている。想定される挙動は以下の通り。

  • 3列目から4列目に移動すると、「City Pittsburgh」のように読み上げる
  • 2行目から1行目に移動したときは、Phone#やFax#とだけ読み、Cityは読み上げない。

最初の挙動だけではthを解釈しているのか、scope属性まで解釈しているのか判断できないが、2番目のように読み上げれば、thではなくscope属性を解釈していることまで判断できるのではないか。
